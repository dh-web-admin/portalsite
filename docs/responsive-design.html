How PortalSite adapts to different viewports and devices

Guide Last updated: November 6, 2025

Viewport Configuration

All pages include a viewport meta tag to ensure proper scaling on mobile devices: 

<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=5.0, user-scalable=yes" />
<meta name="theme-color" content="#667eea" />

Responsive Breakpoints

Design is tuned to several breakpoints to maintain usability across devices.

Desktop (> 1024px)

  • Full sidebar (240px)
  • 3-column dashboard tiles
  • Full header with logo

Tablet (768px - 1024px)

  • Reduced sidebar (200px)
  • 2-column dashboard tiles
  • Smaller header spacing

Mobile Landscape / Tablet Portrait (600px - 768px)

  • Compact sidebar (200px)
  • 2-column tiles or stacked layout
  • Smaller fonts and reduced padding

Mobile Portrait (< 600px)

  • Sidebar hidden behind hamburger menu
  • Full-width tiles (single column)
  • Touch-optimized buttons (44px minimum)
  • Zero left margin on content

Very Small Mobile (< 380px)

  • Further reduced font sizes and minimal padding

Mobile Navigation

Behavior for screens under 600px:

  • Sidebar slides in from left via CSS transform when hamburger is clicked
  • Dark overlay prevents interaction with the page content while menu is open
  • Menu closes on overlay click or when a navigation link is selected
  • Body scroll is locked when menu is open

Responsive Images

  • All images use max-width: 100%; to prevent overflow
  • Maintenance images scale down at defined breakpoints (400px ‚Üí 300px ‚Üí 250px)
  • Consider adding srcset for high-DPI devices in future

Touch-Friendly Elements

  • Touch targets follow minimum 44x44px guideline
  • Form inputs use 16px+ font to avoid iOS zooming
  • Buttons and interactive areas have generous spacing to prevent accidental taps

Flexible Layouts

  • Dashboard tiles use flexbox with wrapping
  • Content areas use percentage widths and max-width constraints
  • Sidebar and main content margins/padding adjust per breakpoint
